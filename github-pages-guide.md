# GitHub Pages Guide

Use GitHub Pages to publish the ActionFlow dashboard.

## Recommended Repository Name

```text
meeting-action-tracker
```

## Enable GitHub Pages

1. Open the GitHub repository.
2. Go to **Settings**.
3. Go to **Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Branch: `main`.
6. Folder: `/root`.
7. Click **Save**.

GitHub will give you a URL like:

```text
https://YOUR_USERNAME.github.io/meeting-action-tracker/
```

## Published Links

Use the GitHub Pages link as the live app link.

Use the GitHub repository link for source code and technical artifacts.

Recommended public links:

```text
Live app: https://hildegarht1.github.io/meeting-action-tracker/
Source code: https://github.com/Hildegarht1/meeting-action-tracker
```

## Final Demo Checklist

```text
1. Open the live app.
2. Enter a meeting title.
3. Paste notes or use Start listening.
4. Click Extract actions.
5. Confirm Make webhook mode if the endpoint is configured.
6. Check the Google Sheets log.
7. Export the CSV.
```

## What Not To Include

Do not publish API keys, OAuth credentials, private webhook URLs, or private Google Sheets links.
